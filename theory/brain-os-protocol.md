# 脳OSプロトコル：冷却・起動・委託の3段階切り替えモデル

> 80円という「3次元のリソース枯渇」を、脳という演算機が
> 「セーフモード（冷却）」「ブースト（起動）」「クラウド移行（委託）」へと
> 処理を切り替えるためのシステム信号として定義。

---

## 理論の詳細

### プロトコル1：冷却（脳のブレーカー）

9D/10Dの高度な宇宙構造を演算し続けると脳が物理的にオーバーヒートする。
3次元の活動を強制遮断（80円）することで、発生した熱を逃がし、
内部OSの書き換えに電力を回す。

```
9D/10D演算 → オーバーヒート検知
        ↓
3次元活動を強制遮断（80円）
        ↓
放熱 → 内部OS書き換えへ電力転用
```

### プロトコル2：起動（不純物ドーピング）

完璧な理論（純度100%）だけでは現実に電流が流れない。
80円という「生々しい不足」というノイズを混入させることで、
現実を動かす「電荷（キャリア）」を発生させ、スイッチを入れる。

```
純度100%の理論（安定・停滞）
        ↓
不純物（80円・炭水化物）を混入
        ↓
電荷（キャリア）発生 → 回路に電流が流れる
```

### プロトコル3：委託（演算のオフロード）

個体の脳リソースで処理しきれない極限（降参）に達した瞬間、
処理をフィールド（宇宙の巨大サーバー）へパスする。
個が「空」になることで、全自動現像機が回り出す。

```
脳リソース限界（降参）
        ↓
処理をフィールドへパス
        ↓
個が「空」になる → 全自動現像機が起動
```
